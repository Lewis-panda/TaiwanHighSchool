# 筆記庫規格（請先讀我）

> [!info] 這份檔案的用途

## 2. 資料夾與檔案結構

```
<課程冊資料夾>/
└── <章代碼 章名>/                ← 一個資料夾 = 一章
    ├── <章代碼 章名>.md          ← 章節「主筆記」（上課主軸）
    ├── <章代碼> 練習題.md         ← 段考用練習題（含可折疊詳解）
    ├── background/               ← 該章的延伸 / why 筆記
    │   ├── 為什麼……？.md
    │   └── ……
    └── sources/                  ← 該章的圖片（.svg）與生成來源
        ├── <章代碼>-<圖名>.svg
        └── ……
```

- **一章 = 一個資料夾**，資料夾名 = 主筆記檔名 = `章代碼 章名`。
- 每章三類產出：**主筆記**（上課主線）＋**練習題**（段考用）＋**why 筆記**（`background/`）；圖片放 `sources/`。
- `sources/` 由繪圖腳本自動建立（見第 9 節），圖檔嵌入用 `![[圖名.svg]]`。

## 3. 命名規則（重要，關係到連結是否會壞）

> [!danger] Obsidian 的 `[[wikilink]]` 是**靠檔名**解析，不是靠路徑
> 全庫**每個 `.md` 檔名必須唯一**，否則連結會指錯。因此：

- **章主筆記**用「章代碼」前綴，全庫唯一：
  - 數學：`數1-x`（第一冊）、`數2-x`（第二冊）、`數A3-x`（數A第三冊）、`數A4-x`（數A第四冊）
  - 物理：`必物-x`（必修物理）、`選物I-x`、`選物II-x`
  - 例：`必物-2 物體的運動.md`、`數A3-1 三角函數.md`
- **why / background 筆記**用**具體、不會撞名的標題**，並在結尾加章代碼以保唯一：
  - 格式：`為什麼……？（章代碼）.md`，例：`為什麼慣性質量會等於重力質量？（必物-2）.md`
  - 標題盡量是一個**真正的問句**，看標題就知道在問什麼。

## 4. Obsidian 語法慣例

### Callout（標註框）
固定用途如下，讓全庫風格一致：

| 語法 | 用途 |
|---|---|
| `> [!abstract] 本章地圖` | 章開頭的重點/架構總覽 |
| `> [!info] 課綱對應` | 標示本節對應的學習內容代碼 |
| `> [!note] 定義` | 嚴謹定義 |
| `> [!example] 例題` | 例題與解法 |
| `> [!tip] 教學提示` | 給老師自己的上課提醒、易錯點、板書建議 |
| `> [!question] 為什麼？` | 丟出 why 問題，並 `→ [[……]]` 連到 background 筆記 |
| `> [!warning] 常見迷思 / 尚無定論` | 學生常見錯誤，或物理界尚未有公認答案之處 |

可折疊：`> [!note]-`（預設折疊）、`> [!note]+`（預設展開）。

### 數學（MathJax）
- 行內：`$F = ma$`
- 獨立式：
  ```
  $$
  \vec{F}_{\text{net}} = \frac{d\vec{p}}{dt}
  $$
  ```
- 一律用標準 LaTeX；向量用 `\vec{}`，需要時用 `\,`、`\text{}` 排版。

### 連結
- 連到別章主筆記：`[[必物-2 物體的運動]]`
- 連到 why 筆記：`[[為什麼慣性質量會等於重力質量？（必物-2）]]`
- 指定顯示文字：`[[必物-2 物體的運動|上一章：運動學]]`

### 圖片
- 一律用**程式化產生的向量圖（SVG）**，不貼網路圖片（版權＋風格一致）。
- 嵌入語法：`![[必物-2-運動圖形.svg]]`（embed 同樣靠檔名解析）。
- 圖檔名規則：`<章代碼>-<圖名>.svg`，全庫唯一，例：`必物-2-自由體圖.svg`、`數A3-1-正弦函數圖形.svg`。
- 圖檔放在該章 `sources/`；產生方式見第 9 節。

## 5. 主筆記（章節）內容規格 —— **完整講義等級**

> [!important] 主筆記 = 可直接上課的逐節講義
> 不是大綱、不是摘要。深度與密度以 **[[必物-2 物體的運動]]** 為基準：老師拿著它就能照講一堂課。

每份章主筆記**必須**包含：

1. **YAML frontmatter**：`科目 / 課程 / 章 / tags`。
2. `# 章代碼 章名`
3. `> [!abstract] 本章地圖`——本章在學什麼、與前後章的關係、為什麼重要、**建議教學節奏**、本章 why 主線。
4. > [!danger] 絕對不要在學生看的筆記裡出現課綱代碼（如 `PKd-Ⅴc-1`、`PEb-Ⅴc-4`）
   > 它干擾閱讀、對學習毫無幫助。章節**依課綱學習內容切分**即可；代碼對照只保留在 `_課綱/`（我規劃用），不寫進任何主筆記、why 筆記、練習題。
5. **依「節」逐節展開的完整教學內容**，每節都要有：
   - **概念引入＋敘事講解**（不只丟定義，要有「為什麼這樣想」的鋪陳）；
   - `> [!note] 定義` 嚴謹定義與**公式推導**（用 `$$`，關鍵結果用 `\boxed{}`）；
   - **多道 `> [!example] 例題`，且附完整逐步解答**（一節至少 1～2 題，計算題要算到底）；
   - `> [!warning] 常見迷思`——學生易錯點、考試陷阱；
   - `> [!tip] 教學提示`——板書順序、演示、生活例子、講多深的建議；
   - 需要圖時用**文字圖說**：`> [!note] 圖：…` 描述，或用程式碼區塊畫簡單 ASCII 示意（自由體圖、v–t 圖、斜面等）。
6. 在課本「只給公式/現象、不解釋」之處，插入 `> [!question] 為什麼？` 並連到 `background/` 的 why 筆記。
7. `## 本章重點整理`——`> [!abstract]` 一頁複習（公式表、關鍵結論）。
8. `## 隨堂練習（附解答）`——用可折疊 `> [!question]-` 收納題目與解答。
9. `## 延伸與 why 筆記`——列出本章所有 background 筆記的連結。
10. `## 與其他章節的連結`——前後章、跨科、數學工具連結。

語言：**繁體中文（台灣用語）**；專有名詞首次出現附英文，例：動量（momentum）。
篇幅參考：一章主筆記約抵 `必物-2`（含 5～8 道例題、數個推導、重點整理、隨堂練習）的份量。

## 6. why / background 筆記內容規格（本庫的靈魂）

目的：把課本「只給結論、沒講清楚」的地方補滿——**從最基本到最前沿都要顧**，三個層次缺一不可：

> [!important] background 必須先有「基本澄清題」
> 最常見的失敗是只寫艱深/開放題，卻跳過學生第一時間就卡住的基本問題。每章 background **先確保基本澄清題夠多**，再放深入題與開放題。
> - **① 基本澄清題（不可略過）**：課本直接搬出、卻沒交代來歷或定義的東西。例：「什麼是黑體？」「$\lambda=h/p$ 這條式子是怎麼想出來的、不是憑空冒出來的吧？」「什麼是功函數？」「為什麼能階的能量是負的？」這類問題最基本、最該優先補。
> - **② 深入題**：用大學觀點把高中含糊處說透。
> - **③ 開放題**：到今天物理界仍無公認答案者，明確標示為開放。

基本澄清題可以寫得短而清楚；深入／開放題照下列結構（見 `_範本/why筆記範本.md`）：

1. `> [!question]` 把問題講清楚（課本只給了什麼）。
2. `## 課本怎麼說` —— 高中課本的處理方式與其侷限。
3. `## 更深一層` —— 用大學普物 / 進階觀點解釋。要**物理上正確**，寧可說「目前理解是…」也不要編造。
4. `## 物理界目前的理解`（適用時）—— 誠實標示**哪些仍是開放問題 / 有不同學派**，用 `> [!warning]`。
5. `## 回到課堂` —— 老師可以怎麼跟學生講（深淺拿捏、要不要講、怎麼類比）。
6. 結尾 `← 返回 [[章主筆記]]`。

> [!warning] 正確性優先
> 寧缺勿錯。涉及前沿/爭議（如量子測量詮釋、慣性起源、時間之箭）要明確標示為開放問題，不要寫成定論。

## 6.5 章節練習題（段考用）

每章另出一份 `<章代碼> 練習題.md`（見 `_範本/練習題範本.md`，標準範例 [[必物-6 練習題]]）。目的：學生段考準備、老師出卷取材。

- **份量**：每章約 **18～25 題**，涵蓋全章各節。
- **題型分區**（仿台灣段考／學測·分科）：
  1. `## 一、單選題（觀念）`
  2. `## 二、單選題（計算）`
  3. `## 三、多選題`
  4. `## 四、計算·非選題`（要寫出過程）
  5. `## 五、挑戰／實驗探究題`
- **每題附可折疊詳解**：題目下方緊接
  ```
  > [!success]- 詳解
  > 答案：(C)。……（過程與說明）
  ```
  用 `-`（預設折疊），方便當自我測驗；老師可一鍵展開。
- **難度標記**：題號後加 `★`（基礎）／`★★`（中等）／`★★★`（挑戰）。
- 計算題數字盡量算得乾淨；常用近似：$E(\text{eV})=1240/\lambda(\text{nm})$、$E_n=-13.6/n^2$ 等。
- 開頭放 `> [!info] 範圍與對應`，標本卷對應的章節與常用常數（**不放課綱代碼**）。
- 主筆記用一行連到它：`> 完整段考練習見 [[<章代碼> 練習題]]`。

當你被指派產出某一「課程冊」或某幾章時：

1. **先讀**：本檔 `規格.md` → 對應的 `_課綱/數學-學習內容.md` 或 `_課綱/物理-學習內容.md` → `_範本/章節主筆記範本.md`、`_範本/why筆記範本.md` → 標準範例章節 `物理/物理一（必修物理）/必物-2 物體的運動/`。
2. 章節資料夾**已建好**（空的）。你只負責**寫入** `<章代碼 章名>.md` 與 `background/` 內的 why 筆記。
3. 每章至少產出 **1 份主筆記 + 1 份練習題（18～25 題含詳解）+ 2～4 份 why 筆記**。why 問題要挑「課本真的沒講清楚」的點（見各章 `_課綱` 條目旁的提示）。
4. 嚴守第 3 節命名規則、第 4 節語法慣例。完工後在 `README.md` 對應清單把該章打勾。
5. 不要改動別人負責的章節；不要動 `規格.md`、`_課綱/`、`_範本/`。

## 9. 繪圖管線（圖片怎麼生）

> [!info] 工具
> Python + matplotlib，輸出 **SVG**（向量、Obsidian 原生、放大不糊、文字轉路徑跨裝置可靠）。環境裝在庫根目錄 `.venv/`（點開頭，Obsidian 自動略過）。

- 共用樣式庫：`_tools/figlib.py`（統一配色、線寬、中文字型、`save_to()` 輸出到各章 `sources/`）。**所有繪圖腳本都 import 它**，確保全庫風格一致。
- 每章一支腳本：`_tools/fig_<章代碼>.py`（例 `_tools/fig_必物-2.py`），裡面每個函式畫一張圖、呼叫 `F.save_to(fig, CH, "必物-2-圖名")`。
- 產生 / 重繪某章的圖：
  ```bash
  .venv/bin/python _tools/fig_必物-2.py
  ```
- 標準範例：`_tools/fig_必物-2.py` 內含四種典型圖（雙軸函數圖、自由體圖、向量分解示意、軌道/幾何示意），新章節照它的寫法擴充。
- 配色語意（見 figlib）：藍=向量/速度/正向力、紅=重力/反向、綠=張力/輔助、琥珀=摩擦/強調。
- 安裝（若換機器，重建環境）：
  ```bash
  python3 -m venv .venv && .venv/bin/pip install matplotlib numpy
  ```

> [!tip] 給 subagent
> 要畫圖時：寫 `_tools/fig_<章代碼>.py`，`import figlib as F`，用 `F.canvas()/F.schematic()/F.arrow()/F.save_to()`；圖名前綴章代碼；跑腳本生成後，在主筆記用 `![[圖名.svg]]` 嵌入。
